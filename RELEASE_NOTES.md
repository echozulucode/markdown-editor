# Release Notes

Curated highlights for each public release of the `@echozedlabs/*` Markdown editor
packages. Per-package detail lives in each package's `CHANGELOG.md`, generated by
[Changesets](https://github.com/changesets/changesets) at release time. Newest
release first.

The five packages (`core`, `codemirror`, `renderers`, `wysiwyg-lexical`, `react`)
are version-locked and release together.

---

## 0.2.0 — 2026-06-07

This release makes hybrid-mode tables editable, renders inline formatting in
preview, and hardens the React wrapper for correctness, security, and a cleaner
API.

### Highlights

Three changes carry most of the value.

- Editable tables in hybrid mode. Edit cells in place, move with Tab, and
  restructure rows and columns from a toolbar or right-click menu.
- Inline formatting in preview. Bold, italics, links, code, and wiki links render
  instead of showing raw Markdown.
- Renderer HTML is sanitized on the hybrid path, closing an asymmetric XSS gap.

### Added

- Inline-editable hybrid tables. Tables in hybrid mode are now an always-on
  editable widget with `contenteditable` cells. Edit in place, use Tab and
  Shift-Tab to move between cells, and restructure from a contextual toolbar or an
  identical right-click menu: insert row above or below, insert column left or
  right, set column alignment (left, center, right), and delete a row, column, or
  the table. Column alignment round-trips. Cell edits commit when focus leaves the
  table; structural changes commit immediately. The styles ship in
  `@echozedlabs/react/styles.css`.
- Inline Markdown rendering in the default preview renderer. Paragraphs, headings,
  list items, table cells, blockquotes, and callouts now render bold, italic,
  strikethrough, inline code, links, and inline images. Links are scheme-checked,
  so dangerous schemes like `javascript:` stay plain text. Underscores inside words
  (`snake_case`) stay literal. Output remains HTML-escaped and DOMPurify-sanitized.
  Exposed as a new `renderInline` export.
- Wiki links in rendered content. `renderInline` turns `[[Target]]` and
  `[[Target|Label]]` into a styled link
  (`<a class="me-renderer-wiki-link" data-wiki-target="...">`), matching what hybrid
  mode already does. Resolve the real destination from `data-wiki-target`. Styling
  ships in `@echozedlabs/react/styles.css`.

### Changed

- Mode switcher restyle. The mode buttons are now a segmented control. The active
  mode is filled with the accent color, inactive modes get a hover surface, and all
  expose a focus-visible ring. Everything is driven by `--me-*` theme tokens, so the
  control follows the host's light or dark theme. The editor does not impose its own
  system theme.
- Responsive tables. The fixed table-level `min-width` is gone. Per-cell min-widths
  drive horizontal scroll, contained within the table, only when a table is
  genuinely too wide. Tables now fit narrow and phone-width containers.
- `MarkdownEditorProps.rendererRegistry` is deprecated in favor of `renderers`. The
  React component only ever read `renderers`. The unused field will be removed in a
  future minor.

### Fixed

- Imperative updates emit `onChange` exactly once. `setMarkdown`, `replaceMarkdown`,
  and `insertMarkdown` previously fired two or three times. React is now the single
  emit source for imperative paths.
- Preview reacts to renderer-registry changes. Swapping the `renderers` registry
  without changing the Markdown updates the preview.
- Preview no longer loops or drops. The preview renders once per document instead of
  re-rendering continuously, which previously showed as a flickering scrollbar. It
  also renders reliably on first mount and on navigation, and it reserves the
  scrollbar gutter.
- Active mode is clamped to `modes`. If `modes` changes so the current mode is no
  longer allowed, the editor falls back to a valid mode instead of rendering no
  surface.
- `getSnapshot()` includes the selection when CodeMirror is mounted.
- Diagnostics reach both channels. They dispatch to `onDiagnostics` and to
  `hostServices.reportDiagnostics`.
- Hybrid table detection no longer drops valid tables. Center-aligned separators
  (`:--:`) and single-column tables (`| --- |`) are recognized again, in step with
  the table-model parser.
- Mermaid error graphic suppressed. Invalid diagrams surface only through the inline
  fallback and a diagnostic. Mermaid's built-in "Syntax error" SVG no longer leaks
  onto the page.
- No duplicate date-picker icon. Date, time, and datetime fields add a custom picker
  button only when the browser lacks a native one, and render it as an icon rather
  than a "Pick" text button.
- Rich Text (WYSIWYG) tables render with grid lines. The `.me-wysiwyg-table*` rules
  were missing from the consumer-facing `@echozedlabs/react/styles.css`. Tables now
  have borders, and header cells use `--me-surface-muted` so they adapt to dark
  themes.

### Security

- Renderer HTML is sanitized in hybrid mode. The preview surface already ran
  renderer output through `sanitizePreviewHtml`, but the hybrid rendered-block
  widget injected the same HTML unsanitized. That left an asymmetric XSS path if a
  custom or host renderer emitted event handlers or `javascript:` URLs.
  `hybridRenderMarkdown` now sanitizes its output, matching the preview path. A
  regression test fails if a renderer's `onerror` handler reaches the editor DOM.

### Upgrade notes

```bash
npm i @echozedlabs/react@0.2.0 react react-dom
```

There are no breaking API changes. Upgrading from `0.1.0` is a drop-in. Two things
are worth doing:

- If you reference `rendererRegistry`, move to `renderers`. The old prop is
  deprecated.
- The new editable-table, wiki-link, and Rich Text table styles ship in
  `@echozedlabs/react/styles.css`. Import it to pick them up.

## 0.1.0 — 2026-06-04

First public release of the `@echozedlabs/*` Markdown editor. A framework-agnostic
core codec owns the canonical Markdown. CodeMirror renders source and hybrid,
Lexical provides WYSIWYG, and the renderers package produces sanitized preview and
diagram output. A React component wraps them with controlled and imperative APIs.
